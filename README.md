Бот - утренний дайджест
Функционал:
1. каждое утро рассылка новостей всем пользователям в определенное время
2. Рассылка Погоды
3. Курс Доллара, российского рубля, белорусского рубля

Идея Рассылки новостей:
Лимит запросов к api newsdata.io - 200 запросов в сутки. 
Было принято решение сделать систему умной отправки доп. пачек новостей пользователям. 
Утром делается запрос к api на первую пачку новостей. Затем всем пользователям рассылается первая пачка новостей за сутки.
Каждый пользователь может вести команду - /digest и получить доп. пачку новостей (1, 2, 3, 4 и тд.)
Обработчик команды пользователя который первый напишет эту команду за доп. пачкой - делает запрос к api новостей, 
получаем вторую пачку, она отправляется в кэш (база данных написанная на ORM peewee), 
и пользователю который написал команду /digest.
Следующий пользователь который напишет эту команду получит вторую пачку уже из кэша, а не из тдельного запроса к api, и т.д.
Таким образом экономится кол-во ежедневных запросов к api.


Структура бота:
файл main.py - запуск базы данных, необходимых утилит, и самого бота
файл config.py - в нем подтягивается токен бота из файла .env в случае его наличия, кортеж базовых команд бота
файл .env (скрыт от внешнего репозитория - gitlab) - необходимые токены для бота
файл .env.template - пример нужных токенов для работы бота
файл .gitignore - указаны файлы и директории которые не должны попасть в удаленный репозиторий gitlab


Папка data (скрыта от внешнего репозитория - gitlab)
содержит - файлы б.д.:
1. subscribers.db - данные о пользователях


Пакет database
содержит:
1. базовый файл __init__.py
2. файл db.py, в нем содержатся классы и функции для работы с базой данных
База данных предствляет собой 3 таблицы: users(информация о пользователях), 
users_news_progress(прогресс просмотра пачек новостей), news_packs(сохраненные пачки новостей)


Пакет handlers
содержит:
1. базовый файл __init__.py
2. файл default_handlers.py в нем обработчики комманд: start_handler() - запуск бота
3. файл news_handler.py в нем обработчики комманд: 
send_news_digest() - обработка команды - /digest (отправка дополнительных новостей)


Пакет parsers
содержит:
1. базовый файл __init__.py
2. Два пакета (виды парсинга): api, html
3. В пакете api: файл news_api c функцией запроса новостей у сервиса api - newsdata.io
4. В пакете html: файл news_parser c функцией запроса новостей 
у обычного российского сайта новостей (риа новости) - не лучший способ (оставил для портфолио)


Пакет utils
содержит:
1. базовый файл __init__.py
2. файл set_bot_commands - устанавливает нужные команды внутри меню приложения telegram
3. пакет scheduler с файлом mailing_digest.py с функциями для рассылки новостей всем подписчикам бота
